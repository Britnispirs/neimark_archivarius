# Telegram-бот «AI-Archivarius»

AI-Archivarius — это умный RAG-ассистент (Retrieval-Augmented Generation, который превращает любые текстовые лекции и учебники в интерактивную базу знаний.

usename telegram: `@ai_archivarius_bot`
## Какую проблему решает и для кого

Для кого: Школьники выпускных классов, студенты и все, кто работает с объемными документами.

Проблема: Огромный объем неструктурированной информации для экзаменов умножается на критическую нехватку времени. Долгий поиск ответов тормозит процесс обучения.

Решение: Бот работает как личный поисковик: мгновенно находит нужный фрагмент в гигантском документе и с помощью нейросети формулирует точный ответ.

## Входные данные и результат

* Входные данные: Текстовые документы (форматы .txt, .pdf, .docx) и текстовые вопросы пользователя к этим документам.
* Результат: Ответ бота, сгенерированный LLM строго на основе загруженного пользователем контекста, без выдуманных фактов.

## Пример работы

1. Пользователь отправляет команду /start и загружает файл `Тихий_дон_отрывок.docx`.
2. Бот парсит файл, нарезает его на блоки по 1000 символов (для обхода лимитов) и сохраняет в векторную БД.
3. Пользователь пишет вопрос по тексту: "В одном предложении расскажи содержание файла".
4. Бот находит 3 релевантных абзаца и выдает краткий, понятный ответ, опираясь только на этот учебник:
 `"В тексте описывается обращение к народу с призывом к действию против Временного правительства, а затем — подготовка и неудачная атака кавалерийских частей на вражеские позиции, где противник отступил, но оставил засады."`.

## Инструкция по запуску

1. Клонирование и настройка окружения

`git clone https://github.com/Britnispirs/neimark_archivarius.git`

`cd neimark_archivarius`

`python3 -m venv venv`

`source venv/bin/activate  # Для Linux/WSL`

`pip install -r requirements.txt`

2. Настройка ключей окружения
Создайте файл .env в корневой папке проекта и добавьте туда ваши личные API-ключи:

`BOT_TOKEN=ваш_токен_от_BotFather`

`MISTRAL_API_KEY=ваш_токен_от_Mistral`

3. Запуск бота
Убедитесь, что виртуальное окружение активировано, и запустите основной скрипт: `python3 main.py`

## Как пользоваться ботом

1. Откройте бота в Telegram и отправьте команду /start (меню команд также доступно в интерфейсе).
2. Загрузите любой учебный материал или лекцию в формате .txt, .pdf или .docx.
3. Дождитесь уведомления об успешном изучении и сохранении документа.
4. Задавайте любые вопросы по тексту обычным сообщением — бот ответит, опираясь строго на загруженные материалы.

## Пример работы

1. **Загрузка материала:** Пользователь отправляет боту команду `/start` и загружает файл `Информатика_Теория.pdf`. 
2. **Индексация:** Бот парсит файл, нарезает его на смысловые блоки по 1000 символов (для обхода лимитов контекстного окна) и сохраняет в локальную векторную базу данных.
3. **Запрос:** Пользователь отправляет текстовый вопрос: *"Как работает алгоритм Дейкстры?"*.
4. **Поиск и ответ:** - Векторная база данных за миллисекунды находит 3 наиболее релевантных текстовых блока, отфильтрованных строго по `user_id` (для защиты приватности).
   - Найденный контекст передается в модель Mistral AI.
   - Бот выдает точный, сжатый ответ, опираясь исключительно на загруженную теорию, без использования сторонней базы знаний.

## Структура проекта

* main.py — Точка входа, инициализация асинхронного ядра (Aiogram 3) и обработчики команд.
* rag_engine.py — Ядро RAG-системы (жесткое чанкование текста, векторизация, логика ChromaDB и запросы к API Mistral).
* requirements.txt — Список внешних библиотек и зависимостей.
* chroma_db/ — Локальная директория векторной базы данных (создается автоматически).
* .env — Флаг переменных окружения с токенами (не включается в публичный репозиторий).

## Инструменты и библиотеки

* Язык: `Python 3.12`
* Фреймворк Telegram: `aiogram 3.x`
* Векторная БД: `ChromaDB` (с изоляцией данных пользователей по метаданным `user_id`)
* LLM-генерация: `Mistral AI` (модель `mistral-tiny`)
* Эмбеддинги (Векторизация): `Sentence-Transformers` (модель `all-MiniLM-L6-v2`)
* Парсинг документов: `pypdf`, `python-docx`

## Роль ИИ в разработке

ИИ применялся в роли технического консультанта. С его помощью были решены следующие архитектурные задачи:

* Проектирование алгоритма обхода лимита контекстного окна (устранение ошибки переполнения 256K лимита при обработке монолитных PDF-файлов).
* Оптимизация процедур парсинга и сжатие громоздких циклов Python в элегантные генераторы списков (list comprehensions).
* Консультирование по вопросам безопасности (реализация жесткой изоляции контекста пользователей в единой коллекции БД для предотвращения утечек).

## Ограничения прототипа

* Слепота к графике: Текущие парсеры извлекают из документов только сырой текст. Анализ сложных формул, схем, графиков или таблиц внутри PDF-файлов не поддерживается.
* Ресурсоемкость: Локальная генерация эмбеддингов требует фонового выделения оперативной памяти, что накладывает ограничения на запуск проекта в бесплатных облачных средах.

## Идеи развития

1. Модуль OCR: Интеграция систем компьютерного зрения для полноценного распознавания текста с изображений и формул.
2. Speech-to-Text: Подключение API Whisper для возможности отправки голосовых лекций, их транскрипции и автоматического пополнения базы знаний.
3. Интервальное повторение: Разработка алгоритма автоматического составления карточек для запоминания на основе изученных ботом документов.
4. Выбор архивов: Функция выбора документа, чтобы пользователь мог конкретизировать ответ бота.
